# How To Run Preview E2E Against Protected Vercel Deployments

This guide explains how to let GitHub Actions run Playwright against protected Vercel preview deployments without exposing Vercel secrets.

## When This Is Required

Use this setup when Vercel preview deployments are protected by Vercel Authentication, Password Protection, Trusted IPs, or other deployment protection settings.

Without a bypass secret, `.github/workflows/preview-e2e.yml` intentionally skips the browser run and emits a GitHub notice. That avoids false failures against Vercel's authentication page and avoids logging protected page contents.

## Secret Flow

Vercel generates an automation bypass secret for the project. GitHub Actions stores the same value as a repository secret named `VERCEL_AUTOMATION_BYPASS_SECRET`.

The workflow passes that secret to Playwright through an environment variable. `playwright.config.ts` sends it only as HTTP headers:

```text
x-vercel-protection-bypass: <secret from GitHub Actions>
x-vercel-set-bypass-cookie: true
```

Do not place the secret in committed files, issue comments, PR bodies, commit messages, screenshots, or copied logs.

## Configure Vercel

1. Open the Vercel project settings for `repo-analyzer-green`.
2. Find Deployment Protection settings.
3. Enable Protection Bypass for Automation.
4. Create a bypass secret for the Preview E2E workflow.
5. Give the secret a note that identifies its purpose, such as `GitHub Actions Preview E2E`.
6. Copy the generated value once for GitHub setup.

If a bypass secret is regenerated or deleted, previous deployments may need to be redeployed before the new value is available for automation.

## Configure GitHub Actions

1. Open the GitHub repository settings.
2. Go to repository secrets for GitHub Actions.
3. Add or update a repository secret named `VERCEL_AUTOMATION_BYPASS_SECRET`.
4. Paste the Vercel-generated bypass secret as the value.
5. Save the secret without copying it into any issue, PR, or local file.

The workflow reads the secret with:

```yaml
VERCEL_AUTOMATION_BYPASS_SECRET: ${{ secrets.VERCEL_AUTOMATION_BYPASS_SECRET }}
```

## Verify

1. Open a PR that triggers a Vercel preview deployment.
2. Wait for Vercel to report a successful preview deployment status.
3. Confirm the `Preview E2E` workflow runs instead of skipping with the missing-secret notice.
4. Confirm the Playwright check passes against the preview URL.
5. Confirm the workflow logs do not include the secret value.

Do not verify by printing the secret. If the workflow still skips, check that the GitHub secret name is exactly `VERCEL_AUTOMATION_BYPASS_SECRET` and that the Vercel preview deployment is reporting an environment URL.

## Rotate

Rotate the bypass secret when access should change or if exposure is suspected.

1. Create or regenerate the bypass secret in Vercel.
2. Update the GitHub Actions repository secret with the new value.
3. Redeploy previews as needed so Vercel recognizes the current automation value.
4. Re-run Preview E2E.
5. Revoke any old bypass secret that should no longer work.
